# Blackjack Table
Human-playable blackjack, backed by a Flask API, Python game engine adapted from [blackjack-ml](https://github.com/chrisfox9158/blackjack-ml), and HTML/CSS/JS frontend interface. Built primarily as a personal-learning introduction to JavaScript and server-client system implementation.

## Stack
- **Engine** — Python game engine, adapted from `blackjack-ml` rules logic as a request/response state machine across per-round phases of blackjack play (`BETTING`, `DEALING`, etc.).
- **Backend** — Flask API-style routing system, based primarily on GET and POST requests to Python engine.
- **Frontend** — Vanilla JavaScript (no framework; entirely fetch-based), supported by HTML structure with CSS styling. SVG card assets rendered via `<use>`.

## Authorship
Engine, API, and majority of front-end logic are hand-written. In the frontend CSS and animation layer, a portion of code was generated by Claude; extensively reviewed and debugged by hand.

## Third-Party Assets
Playing card graphics are from [SVG-cards](https://github.com/htdebeer/SVG-cards) by Huub de Beer, based on original work by David Bellot, licensed under [LGPL-2.1](static/svg-cards-LICENSE.txt). The asset file (`static/svg-cards.svg`) is used unmodified.

## Setup
```bash
uv venv
uv sync
```

## Usage
```bash
uv run app.py
```

Open `http://127.0.0.1:5000` in a browser. Gamestate is hosted in a single in-memory instance. Restarting the server will reset the game state; however, the state is persistent across page refresh.
